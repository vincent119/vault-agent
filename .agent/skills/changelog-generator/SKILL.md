# 更新日誌產生器 (Changelog Generator)

這項技能可以將技術性的 git commits 轉換為經過潤飾、讓客戶與使用者都能輕鬆理解並感到滿意的更新日誌。

## 何時該使用此技能

- 準備新版本的發布說明 (Release Notes)
- 建立每週或每月的產品更新摘要
- 為客戶記錄系統變更
- 撰寫提交至 App Store 的更新日誌
- 產生更新通知
- 建立內部發布文件
- 維護公開的產品更新頁面 / 更新日誌

## 這項技能的功能

1. **掃描 Git 記錄**：分析特定時間段或版本之間的 commits。
2. **變更分類**：將 commits 分門別類（例如：新功能、改善項目、錯誤修正、重大變更、安全性更新）。
3. **翻譯技術轉化為使用者語言**：把開發者的技術 commit 訊息轉換為客戶易於理解的文字。
4. **專業排版**：建立乾淨、結構化的更新日誌條目。
5. **過濾雜訊**：排除內部與開發相關的 commits（如重構、測試等）。
6. **遵循最佳實務**：套用更新日誌準則以及符合您的品牌語氣。

## 如何使用

### 基本用法

從您的專案儲存庫中：

```bash
從上一個版本之後的 commit 建立更新日誌
```

```bash
為過去一週內的所有 commit 產生更新日誌
```

```bash
建立 v2.5.0 版本的發布說明
```

### 指定日期範圍

```bash
為 3 月 1 日至 3 月 15 日之間的所有 commit 建立更新日誌
```

### 套用自訂準則

```bash
為 v2.4.0 之後的 commit 建立更新日誌，並使用 CHANGELOG_STYLE.md 中的更新日誌準則
```

**使用者**：「為過去 7 天的 commit 建立更新日誌」

## 提示

- 請從您的 Git 儲存庫根目錄執行。
- 可以指定日期範圍以聚焦特定的更新日誌。
- 使用您的 CHANGELOG_STYLE.md 以保持一致的格式。
- 在發布之前，請預覽並調整產生的更新日誌。
- 可以直接將輸出結果儲存到 CHANGELOG.md。

## 更新日誌產生器 (Changelog Generator)

此技能負責將技術性的 Git Commit 訊息轉換為：

- 使用者可理解的更新說明
- 符合 Semantic Versioning 的版本結構
- 結構清晰、可直接發布的 Release Notes
- 可用於 GitHub Release / App Store / 官網公告 / Email 通知

---

## 核心原則

1. 不可直接複製 commit 訊息
2. 必須轉換為「使用者價值導向描述」
3. 過濾所有內部與開發雜訊
4. 輸出格式必須一致且可預測
5. 僅保留對最終使用者有影響的變更

---

## 版本規範

- 必須遵循 Semantic Versioning（MAJOR.MINOR.PATCH）
- 每個版本區塊格式如下：

## [X.Y.Z] - YYYY-MM-DD

---

## 變更分類（固定分類，不可自行擴充）

- ✨ 新功能 (Added)
- 🔧 改善項目 (Changed / Improved)
- 🐛 錯誤修正 (Fixed)
- ⚠️ 重大變更 (Breaking Changes)
- 🔐 安全性更新 (Security)

---

## Commit 對應規則

feat → ✨ 新功能
fix → 🐛 錯誤修正
perf → 🔧 改善項目
refactor → 過濾（除非影響使用者）
docs → 過濾
test → 過濾
chore → 過濾

---

## 雜訊過濾規則

必須排除：

- CI/CD 變更
- 版本號調整
- 重構
- 單元測試
- 註解修改
- 日誌調整
- 套件升級（若無使用者影響）
- 內部重命名
- 結構優化（無功能變更）

---

## 撰寫規則

1. 一個 bullet 僅描述一個邏輯變更
2. 使用完成式與肯定語氣
3. 不可出現：
   - commit hash
   - PR 編號
   - 內部模組名稱
   - 技術堆疊細節
4. 不可使用模糊字眼（improve / update / refactor），除非明確說明使用者影響
5. 強調「解決了什麼問題」或「帶來什麼好處」
